# Changelog

All notable changes to Cairn are recorded here. Versions are sequential integer build numbers (one increment per `release.ps1` run).

This Changelog starts at the **major rewrite that replaces the legacy 1.x series.** The rewritten libs ship under the same `Cairn-<Name>-1.0` LibStub MAJORs (`Cairn-Addon-1.0`, `Cairn-DB-1.0`, etc.). The Cairn-Gui family keeps its `-2.0` MAJORs unchanged (`Cairn-Gui-2.0`, `Cairn-Gui-Widgets-Standard-2.0`, etc.) since the v2 family was ported verbatim, not rewritten.

The format is loosely based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [47] - 2026-05-13

Bug-fix release. Five fixes across Cairn-Gui-2.0 Core, Cairn-Gui-Widgets-Standard-2.0, Cairn-DB, and Cairn-Addon, all surfaced while building Forge_AddonManager. Most-impactful: Cairn-Gui-2.0's `SetPrimitiveShown(false)` now survives state transitions (icons hidden at OnAcquire stayed hidden after the first hover instead of reappearing), and Cairn-DB now rebinds stale `_sv` references on `ADDON_LOADED` so file-scope `DB:New` calls survive WoW's SavedVariables restore.

### Fixed

**`Cairn-Gui-2.0`** (Core MINOR 20 -> 21)

- `SetPrimitiveShown(false)` now persists across state transitions. Before this fix, `applyTextureSource` (called on every hover / press / leave) ended with an unconditional `texture:Show()`, re-revealing any icon that had been hidden via `SetPrimitiveShown(false)` at OnAcquire. Symptom: Cairn-Gui Checkbox's check glyph reappeared on first hover, looking like an automatic "preview on hover" feature but actually a bug. Now `SetPrimitiveShown` sets `rec._userHidden` and `applyRecord` for icons calls `Hide()` after `applyTextureSource` when the flag is set. Any consumer using `DrawIcon` + `SetPrimitiveShown(false)` benefits, not just Checkbox.

**`Cairn-Gui-Widgets-Standard-2.0`** (MINOR 11 -> 12)

- `Checkbox.OnAcquire` now calls `frame:RegisterForClicks("AnyUp")`, matching the Button MINOR=4 fix. Without this, the Primitives layer's `OnMouseDown` / `OnMouseUp` HookScripts swallow `OnClick` on Interface 120005 and the checkbox never toggles. Consumers saw the row hover background tint the empty box and assumed it was a visual quirk; the actual problem was clicks never reaching the handler.
- `TabGroup.OnAcquire` explicitly clears `pane.Cairn._builtOnce` after acquiring each per-tab content Container. Pooled Containers can carry consumer-set fields from a previous owner; without the clear, a recycled pane carried a stale `_builtOnce = true` and the new consumer's first-time-build gate fell through.

**`Cairn-DB`**

- File-scope `DB:New` calls now survive WoW's SavedVariables restore timing. Previously a DB instance created at file scope (before `ADDON_LOADED`) bound to an empty pre-emptive table that WoW later orphaned when restoring `_G[svName]`. Persisted data appeared to not survive `/reload`. Added an `ADDON_LOADED` + `PLAYER_LOGIN` listener that walks `Cairn_DB.instances` and rebinds stale `_sv` references in place. Consumer-captured references stay valid; the rebind touches the lib-internal pointer set.
- `:New` stale-cache check drops a cached instance whose `_sv` no longer matches `_G[name]`, covering the `/reload`-restored-SV path.

**`Cairn-Addon`**

- `AddonDBName` now derives from `tocName` (folder name) instead of TOC `Title`. Titles often contain spaces or different casing than the `## SavedVariables:` declaration; deriving from folder name lines up reliably. `opts.dbName` override is available when the SV name diverges from the folder name on purpose.


## [46] - 2026-05-12

Large feature pass across every rewritten lib. New Cairn-Settings surface for EditMode-managed frames (declarative `:Add(frame, schema)`), Blizzard-frame override family, spec-aware-profile API, three storage backends, sub-settings, more. Cairn-Locale gains an AceLocale-style `:NewLocale` write-proxy with auto-key-as-value. Cairn-Hooks adds an AceHook-style API surface alongside the existing Pre/Post/Wrap. Cairn-Timer gains string-method dispatch, fps-drift compensation, variadic args, and caller-id profiling. Cairn-Slash adds `:RegisterChatCommand` per-embed registry + locale-fallback subcommand matching. Cairn-Events grows a separate messages registry with auto-namespace by addon tag. Cairn-DB auto-shrinks SavedVariables on logout.

### Added

**`Cairn-Settings`** (MINOR 14 → 24)

- `:Add(frame, schema, opts?)` registers an addon-owned frame into Blizzard's EditMode with a declarative widget schema. Widget descriptors carry `kind/key/label/desc/default/get/set/disabled/disableif/hidden/tags`. Adding a new widget kind is a registry entry in `Cairn_Settings.editModeKinds`, not a new method. Built-in kinds: `anchor`, `hideable`, `secureFrameHideable`, `scale`, `size`, `alpha`. Allocates a system ID via `:AllocateCustomSystemID`, stamps `frame.system`, materializes `framesDB[frame]` + `framesDialogs[frame]`, records in `_frames`. Idempotent on frame.
- `:AddSystemSettings(systemID, schema, subSystemID?)` extends Blizzard's stock EditMode dialogs (action bars, raid frames) with consumer-defined widgets.
- `:OverrideBlizzardFrame(frame, point, relativeTo, relPoint, ox, oy, opts?)` programmatically relocates Blizzard's EditMode-managed frames without `/reload` or taint. Auto-creates a Character-scoped `"<AddonName> Overrides"` layout when the active layout is an immutable Preset.
- `:SetBlizzardFrameSetting(frame, settingEnum, value)` writes non-anchor properties (scale, alpha, size) with min/max/type validation against `EditModeSettingDisplayInfoManager`.
- `:BeginBatch()` / `:ApplyChanges()` / `:SaveOnly()` — coalesce multiple override calls into a single EditMode refresh. Combat-aware: `:ApplyChanges()` queues for `PLAYER_REGEN_ENABLED` if combat-blocked, auto-flushes on combat exit.
- `:RegisterTaintClearance(taintTarget, clearanceFn)` + `:IterateTaintClearances()` — documented recipes for clearing specific Blizzard-API taint states. Ships the `DropDownList1` recipe (show/hide `AddonList`) as the seed entry.
- `:EnhanceDB(db, friendlyName?)` ports LibDualSpec-style spec-aware-profile capability natively. Mixes 6 spec-aware methods onto the db via `rawset` (no wrapper handle): `:IsSpecProfileEnabled`, `:GetSpecProfile(spec?)`, `:SetSpecProfile(name, spec?)`, `:SetSpecProfileEnabled(bool)`, `:CheckSpecProfileState`, `:RewireDeletedProfile(deletedName)`. One lib-owned listener fires on `PLAYER_SPECIALIZATION_CHANGED` / `ACTIVE_TALENT_GROUP_CHANGED` and walks a weak-keyed registry.
- `:EnhanceOptions(options, db)` injects spec-profile UI via `options.plugins["Cairn-Settings-SpecProfile"]` without mutating `options.args`. Strings route through Cairn-Locale when loaded; English fallbacks inline.
- `:IterateDatabases()` iterator yields `(db, friendlyName)` over enhanced-DB registry.
- Schema entry gains `storage` field with three backends: `"addon"` (default — db.profile), `"cvar"` (WoW CVar system via `GetCVar` / `SetCVar` against `entry.cvar`; toggle entries auto-translate "1"/"0"), `"proxy"` (consumer-supplied `getValue` / `setValue` closures). Default-seeding gated on `storage == "addon"`.
- Schema entry gains `subSettings` array for visually-nested child entries with auto-locking on parent value via `SetParentInitializer`. Children flatten into `_byKey` for Get/Set/OnChange access. Override the parent-lock predicate via `entry.subSettingsModifiable = function(get) -> bool`.
- Schema entry gains `disableif = function(get) -> bool` for reactive per-widget enable/disable, re-evaluated after every `setValue`. `disabled = bool` for static-disable.
- Schema entry gains `tags = {string}` for Blizzard panel search bar (`initializer:AddSearchTags`). Label auto-added.
- Schema entry gains `namePhraseId` / `descPhraseId` resolved through Cairn-Locale at panel-build time.
- Schema entry gains `isVisible(fn)` / `canSearch(bool|fn)` / `newFeature(bool|fn)` runtime predicates.
- `:New(addonName, db, schema, opts)` accepts an optional 4th opts arg. `opts.layout = "vertical"` (default) or `"canvas"`; canvas registers a consumer-owned panel frame and skips schema-based render.
- `:GetWidgetsByType(kind)` returns `{entry, setting, initializer}` for every schema entry of the given kind.
- `:OnDisableStateChanged(fn)` subscribes a renderer to disableif state transitions (fires on change only).
- `:RegisterControl(name, spec)` extension point for consumer-defined control types. Spec carries `storageOnly`, `skipDefault`, `requireArguments` shape validator, optional `validator(entry)` predicate.
- `:OpenToCategory(addonName, subName?)` deep-links to a registered Blizzard Settings panel.
- `:ModifiedClickOptions(mustChooseKey)` returns ALT/CTRL/SHIFT/(NONE) dropdown fragment.
- `:RegisterCustomSetting(name, settingType)` allocates IDs from 100+ (0-99 reserved for Cairn built-ins). `Cairn.Settings.CUSTOM_SETTING_CONSUMER_FLOOR = 100` as a public constant.
- `:AllocateCustomSystemID()` returns a fresh integer system ID seeded from `#Enum.EditModeSystem`.
- `:NormalizeAnchorByQuadrant(frame)` picks the screen corner closest to the frame's center and anchors there with signed offsets relative to UIParent. Keeps movable frames stable across UIScale + resolution changes.
- `:FireHideDialog(senderTag)` cross-addon dialog coordination via `EventRegistry:TriggerEvent("Cairn.Settings.hideDialog")`.
- `:ResolveLayoutProfileName(layoutType, layoutName)` + `:LoadLayoutProfile(db, layoutType, layoutName)` per-EditMode-layout persistence. Account-scope returns `<layoutType>-<layoutName>`; Character-scope appends `-<character>-<realm>`.
- `:InstallEditModeHooks()` (called automatically on first `:Add`) installs `hooksecurefunc` side-handlers on `EnterEditMode` / `ExitEditMode` that force-show hidden registered frames during configuration and restore their predicate-driven hidden state on exit.
- `hookVersion` lib-level field stamped at load. All hooks gate on `hookVersion == MY_MINOR` at fire-time so lib upgrades self-deactivate stale hooks.
- Compound control kinds (`checkbox_and_slider`, `checkbox_and_dropdown`, `checkbox_and_button`) and `lib_shared_media_dropdown` accepted by the schema. Visual rendering for these is deferred to a future build pending Blizzard template feature-detection + LSM preview-frame plumbing; consumers can use `subSettings` or `:RegisterControl` today.
- Defensive `__index` metatables on `framesDB` and `framesDialogs` auto-create entries with a `settings = {}` sub-field on first access.

**`Cairn-Locale`** (MINOR 14 → 16)

- `:NewLocale(app, locale, isDefault, mode)` AceLocale-style write-proxy. Default-locale form accepts the auto-key-as-value shorthand (`L["Hello"] = true` → `L["Hello"] = "Hello"`) and refuses overwrites of existing keys (first-definition wins; explicit `nil` still deletes). Non-default-locale form requires explicit strings.
- Three missing-key modes via 4th arg: `"warn"` (default — prints once per missing key), `"silent"` (returns key, no print), `"raw"` (returns nil).
- Returns nil for non-current non-default locales by default so the locale file can `if not L then return end`. Set `Cairn.Locale.devMode = true` before any locale loads to populate every bank (translator support).
- `:SetActiveLocale(locale)` alias for `:SetOverride`.
- `GAME_LOCALE` global takes precedence over `GetLocale()` in `:GetLocale()`. Resolution order is `_override` → `GAME_LOCALE` → `GetLocale()` → `"enUS"`.
- `:GetPhrase(name, key)` lib-level lookup that returns nil on total miss (vs instance `:Get` which returns the key). `:GetEnglishFallback(name, key)` reads the enUS bank regardless of current locale.

**`Cairn-Slash`** (MINOR 14 → 17)

- `:RegisterChatCommand(target, command, fn, persist)` flat slash registration with per-embed tracking. `:UnregisterChatCommand(target, command)` and `:OnEmbedDisable(target)` for cleanup. `persist = true` keeps the slash alive across `:OnEmbedDisable`.
- `:RegisterSubcommand(localeKey, handler, opts)` registers a sub-command matching BOTH the current-locale form AND the English form via Cairn-Locale. Non-English users can follow English-language guides verbatim. `opts.strictLocale = true` opts out. Case-insensitive dispatch via parallel `_subsLower` map.
- `:GetArgs(str, numargs)` parser preserves quoted strings, Blizzard hyperlinks (`|H...|h[Display]|h`), and texture escapes (`|T...|t`) as single tokens.
- `:Register(name, slash, opts)` accepts `opts.addonName` to override the Cairn-Locale instance name for locale-fallback lookups.

**`Cairn-Hooks`** (MINOR 14 → 16)

- AceHook-style API alongside the existing Pre/Post/Wrap family: `:Hook` (non-secure replacement with auto-chain, errors on secure miss-use), `:SecureHook` (hooksecurefunc-style; side-handler fires after original), `:RawHook` (replacement without auto-chain; receives original as first arg), `:FailsafeHook` (handler pcall'd, original always fires).
- `actives[uid]` soft-unhook system. `:Unhook(handle)` sets the flag so the wrapper no-ops on subsequent fires. `:UnhookAll(owner)` batch-clears every uid under that owner.
- `:HookOnce(frame, script, callback)` installs one real `HookScript` and fans out to all subscribers, then wipes the list. `:HookAlways(frame, script, callback)` is the persistent variant. `:HookFuncOnce(table, methodName, callback)` is the `hooksecurefunc` analogue with fire-once semantics.

**`Cairn-Timer`** (MINOR 14 → 16)

- `fn` accepts a string method name OR function reference. When `fn` is a string, the lib dispatches as `opts.obj[fn](opts.obj, args...)`. Saves closure allocation and honors late-binding.
- Variadic args after `opts` with `argsCount = select("#", ...)` so nil holes are preserved. `CT:After(0.5, fn, opts, "a", nil, "c")` correctly delivers three args including the middle nil.
- fps-drift compensation on `:Every` timers. Each tick records `_expectedFireAt`; the next tick's delay subtracts how late the previous tick fired so average frequency stays accurate across long sessions and alt-tab pauses.
- `opts.from` caller-id tag + `Cairn.Timer.debugMode` flag + `:GetCountAfter()` / `:ResetCountAfter()` accessors for attribution profiling. Counter logic short-circuits when debugMode is false (production default).
- `:ContinueAfterCombat(fn)` fires fn immediately if out of combat, otherwise queues for `PLAYER_REGEN_ENABLED`. One shared event frame across all consumers.
- `:Start(slot, mode, period, fn)` unifies four canonical debounce/throttle patterns: `push` (trailing-edge debounce), `ignore` (leading-edge throttle), `duplicate` (fire-and-forget), `cooldown` (leading-edge with trailing-merge).

**`Cairn-Events`** (MINOR 14 → 16)

- Separate `_messages` registry alongside the unified `handlers` table. `:SubscribeMessage(name, handler, target?, owner?)` / `:UnsubscribeMessage(sub)` / `:SendMessage(name, target, ...)`. Bare names auto-prefix with the consumer's `tocName` resolved from `target` (string tocName, Cairn-Addon Metadata table, or addon namespace table). Cross-addon subscribers reach foreign messages via the fully-qualified name.
- `:Once(event, handler, owner)` and `:OnceMessage(event, handler, owner)` one-shot listeners.
- `:Fire` and `:SendMessage` log to Blizzard's `/eventtrace` UI via `EventTrace:LogCallbackRegistryEvent`. Zero overhead when `/eventtrace` isn't open.
- `:ValidateEvent(eventName)` pre-flight check for typo detection via a module-scope throwaway frame.
- `:SubscribeUnit(event, unit, handler, owner)` with unit-token validation against known unit-prefix list. Handler only fires when the event's first arg matches the bound unit.
- `:IsUnitEvent(event)` detection helper.
- `:UnsubscribeOwner(owner)` walks both `handlers` AND `_messages`.

**`Cairn-DB`** (MINOR 14 → 16)

- 8-bucket partition model: `global`, `profile`, `char`, `realm`, `class`, `race`, `faction`, `factionrealm`. Identity captured at lib load via `UnitName / GetRealmName / UnitClass / UnitRace / UnitFactionGroup`.
- Wildcard defaults via metatable `__index`. `["*"]` applies the sub-table as the default for every direct child; `["**"]` applies recursively at every depth. Lib-internal `_`-prefixed keys skipped.
- `:RegisterMigration(version, fn)` framework. Walks `internalVersion+1 → max(registered)` in order; throwing migrations route to `geterrorhandler`. `Cairn_DB.MIGRATION_DEFER` sentinel queues a migration for `PLAYER_LOGIN` retry; second defer errors loudly.
- `:RegisterNamespace(MAJOR)` returns a sub-DB with its own 8-bucket partition. Idempotent on MAJOR.
- `db:_RemoveDefaults()` walks every registered instance at `PLAYER_LOGOUT` and strips per-key values matching the consumer's defaults; on next load the wildcardMerge re-fills the stripped keys. SV file shrinks. Blocker rule: a table-typed default does NOT recurse into a non-table user value (prevents data corruption when the user wrote a scalar where the default is a table). Lazy listener installed only when at least one instance carries defaults.

**`Cairn-Log`** (MINOR 14 → 15)

- Python-style 6-level scheme with gaps: `TRACE=0, DEBUG=10, INFO=20, WARNING=30, ERROR=40, FATAL=50`. `WARN` preserved as a numeric alias for `WARNING`. Gaps reserve room for consumer-defined intermediate levels.
- Compact `{t, s, m}` entry-shape aliases via metatable `__index` on every pushed entry. Full `{timestamp, source, category, level, message}` shape unchanged.
- Opt-in database backing via `:SetDatabase(svTable)` and optional `db` arg on `:New(name, db?)`. Ring buffer entries also append to the SV table, bounded by the same capacity.
- `:ForceError(fmt, ...)` and `:ForceFatal(fmt, ...)` bypass the chat-echo threshold to guarantee delivery.
- `:SetPerformanceMode(threshold)` nils method slots below the threshold. Consumers gate hot calls via `if Cairn.Log.hasDebug then myLog:Debug(...) end`. Boolean flags `hasTrace` / `hasDebug` / `hasInfo` / `hasWarning` / `hasError` / `hasFatal` maintained in lockstep.
- `:Embed(target, name)` mixin injects `:Info`, `:Debug`, `:Warn`, `:Error`, `:Category` onto target.

### Fixed

- `Cairn-Media-1.0` `:GetQualityColor(name)` strips the `|cff` prefix from `ITEM_QUALITY_COLORS[i].hex`. Blizzard's `.hex` field includes the color-escape prefix (10 chars total: `|cffAARRGG`). Returning that verbatim made `Cairn.Util.String.Colorize(text, hex)` produce double-prefixed `|c|cff...|r` malformed strings. Now returns the raw 8-char AARRGGBB form matching the other Get*Color methods.

### Fixed

- **`Cairn-Media-1.0` `:GetQualityColor(name)` strips `|cff` prefix from `ITEM_QUALITY_COLORS[i].hex`.** Blizzard's `.hex` field includes the color-escape prefix (10 chars total: `|cffAARRGG`). Returning that verbatim made consumer code using `Cairn.Util.String.Colorize(text, hex)` produce double-prefixed `|c|cff...|r` malformed strings. Now returns the raw 8-char AARRGGBB form matching `:GetExpansionColor` / `:GetFactionColor` / `:GetThresholdColorHex` conventions. Caught by build 19 smoke; fixed in build 20. Smoke for the below-range threshold-clamp boundary case also fixed (was using ascending stops where below-range = WORST not best; corrected to descending stops where the assertion makes sense).

### Added

- **`Cairn-Media-1.0` color helpers.** Four new methods on top of the existing dual-bucket asset registry: **`:GetExpansionColor(name)`** — case-insensitive lookup against a 10-entry private palette covering Classic through Dragonflight, returns 8-char AARRGGBB hex or nil. Absent entries return nil so consumers can default-color or skip. **`:GetQualityColor(name)`** — thin name-based wrapper around WoW's `ITEM_QUALITY_COLORS` globals. Maps "Poor" / "Common" / "Uncommon" / "Rare" / "Epic" / "Legendary" / "Artifact" / "Heirloom" / "WowToken" (case-insensitive) to the Blizzard-global `.hex` value. Returns nil for unknown names. Cairn-Media doesn't hardcode hex values — Blizzard's globals stay the source of truth. **`:GetFactionColor(name)`** — same shape as GetQualityColor but for `FACTION_BAR_COLORS`. Reads `.r/.g/.b` floats and formats to AARRGGBB. Includes hardcoded fallback hexes (`FF4A54E8` Alliance, `FFE50D12` Horde) for Classic Era clients where the global isn't present. **`:GetThresholdColor(value, stops...)` + `:GetThresholdColorHex(value, stops...)`** — multi-stop value-to-color gradient interpolator on a green-yellow-red gradient. Variadic threshold stops accept ascending OR descending order. NaN guard via `value ~= value`, Inf guard via `math.huge` check, both fall to neutral yellow. Out-of-range values clamp to the nearest stop.

- **`Cairn-Slash-1.0` `:GetArgs(str, numargs)`.** Lib-level helper that splits a string into `numargs` args plus a remaining tail, preserving three non-trivial cases that naive whitespace-split corrupts: quoted strings (`"Cathedral Square"` → one arg), Blizzard hyperlinks (`|H...|h[Display]|h` — item / quest / achievement / spell / encounter / glyph / etc., all sharing the same envelope), and texture escapes (`|T...|t`). Returns `(arg1, arg2, ..., argN, remaining)` where missing args are nil and the remaining tail has leading whitespace trimmed. Handles unmatched quotes (consumes the rest as the token) and malformed hyperlinks/textures (whitespace-split fallback).

- **Cross-lib small additions.** **`Cairn-Addon-1.0`**: `AUTO_WIRE_FLAGS` table extended with a `Log` entry. The auto-wire spec format gained an optional `attach(addon, lib, tocName)` callback so libs like Cairn-Log that inject methods directly onto the addon namespace via `:Embed` (rather than stashing a reference) can do so transparently. Standard libs (Gui / Slash / Events / Hooks / Timer / Locale / DB / Media) continue to use the default `addon[flag] = lib` attach. `Cairn.Register` now passes `tocName` through `applyAutoWiring`. **`Cairn-Util-1.0`**: `Cairn.Util.SafeMixin(target, ...)` — non-overwriting Mixin with a configurable taint blocklist (initial entry: `AddSnappedFrame`). Only sets MISSING keys; uses `rawget` so metatable defaults don't count as "already set." Returns target for chain ergonomics. Exposed via `Cairn.Util.SafeMixinBlocklist` so consumers can audit / extend. `Cairn.Util.String.Colorize(text, hex)` — wraps text in WoW's `|c...|r` color escape with a caller-supplied 8-char hex. `Cairn.Util.String.ColorizeRGB(text, r, g, b | colorTable)` — same as Colorize but accepts either three (r,g,b) floats in [0,1] OR a single `{r,g,b}` color table matching `RAID_CLASS_COLORS` / `FACTION_COLORS` shape.

- **`Cairn-Addon-1.0` orchestrator API.** Cairn-Addon now exposes a single-call orchestrator on the `_G.Cairn` namespace alongside the existing `:New` lifecycle wrapper. **`Cairn.Register(tocName, Addon, opts) -> Metadata`** chains four phases: (1) TOC metadata extraction into a 12-field Metadata table (`AddonName`, `AddonNameWithSpaces`, `Version`, `NominalVersion`, `Dependencies`, `OptionalDeps`, `IconTexture`, `AccentColor`, `AddonDBName`, `AddonOptionsSlashCommand`, `AddonTooltipName`, `AddonNameWithIcon`), (2) idempotent Cairn-Addon lifecycle instance creation via the existing `:New(tocName)`, (3) auto-creation of a Cairn-DB instance + a minimal Cairn-Settings panel (version header + optional "hide minimap" toggle when `opts.minimap` is truthy; consumer's `opts.options` schema wins when supplied), (4) companion-lib auto-wiring via `opts` flags (`Gui`, `Slash`, `Events`, `Hooks`, `Timer`, `Locale`, `DB`, `Media` — `Settings` deliberately excluded because the return value claims `Addon.Settings`). Re-registration on the same tocName returns the existing Metadata, not a new one. **`Cairn.GetRegistry()`** returns a shallow copy of the per-tocName registry so consumers can iterate / diff / display without mutating the live state. **`Cairn.NewLibrary(name, version, opts)`** wraps `LibStub:NewLibrary` and folds cross-cutting setup (`SetCurrent` to stash at `Cairn.CurrentLibrary`, `MountAs` to assign onto the namespace for non-`Cairn-X-1.0` MAJORs) into one call; the returned lib carries an injected `:NewSubmodule(subName, version, parent?)` method that does TWO assignments (LibStub entry at `<parentMajor>_<subName>` + parent-table assignment at `parent[subName]`). **`Cairn.CurrentLibrary`** auto-tracks the last lib registered with `SetCurrent = true` so a submodule file in the same multi-file lib reads it without hardcoding the parent MAJOR. Calling `NewSubmodule` with no `CurrentLibrary` AND no explicit parent override throws a specific "Check TOC order" diagnostic. **Cairn self-registers** at end-of-file via `C_Timer.After(0, ...)` so Cairn-DB / Cairn-Settings have loaded by the time the registration call runs; the `_internal = true` opt gates the DB/Settings/auto-wiring phases so the self-entry stays diagnostic-only. Re-implements the KLib / AceAddon pattern natively against Cairn-DB and Cairn-Settings; does NOT depend on Ace3. **SavedVariables note for consumers:** addons that call `Cairn.Register` get a Cairn-DB instance auto-created at `<AddonName>DB`; persistence requires `## SavedVariables: <AddonName>DB` in the consumer's TOC.

- **`Cairn-Util.String.ParseVersion(s)` and `Cairn-Util.String.NormalizeVersion(s)`.** Two new String sub-namespace helpers used by `ExtractMetadata` internally. `ParseVersion` extracts the first integer from a version-shaped value (`"2.4.1"` → `2`, `"v3-beta"` → `3`, `nil` → `nil`); accepts both strings and numbers via `tostring`. `NormalizeVersion` produces display-friendly version strings handling three input shapes: clean versions pass through, BigWigs unsubstituted placeholders (`@.+` match) → `"Developer Build"`, SVN keyword `$Revision: N $` → `N`, nil/empty → `"Unknown"`. The load-bearing rule (`s:gsub("@.+", "Developer Build")`) catches BigWigs packager placeholders in dev checkouts. Reference: LibAboutPanel-2.0.

- **`Cairn-Util.ResolveProviderMethod(addon, providerField, methodName)`** top-level helper. Returns a self-bound closure equivalent to `addon[providerField]:methodName(...)`. Built for declarative config tables that reference handler methods by string name rather than by direct function reference — string lookup defers binding to use time, sidestepping file-load-order constraints. Loud errors on miss (missing provider field, non-function method): typo protection. Used by `Cairn.Register` to resolve `opts.minimap.onClick` / similar declarative paths.

### Changed

- **`Cairn-Core` extracted from `Cairn-Util`.** The `_G.Cairn` namespace bootstrap now lives in root-level `Core.lua`, registered as `Cairn-Core-1.0` at MINOR=1 (new lib, conventional starting MINOR). Cairn-Util becomes pure helpers with no global side effects; MINOR stays at 14 (the umbrella-extraction change didn't add or remove consumer-visible API surface). Load order in all 5 flavor TOCs is now `LibStub → Core.lua → Cairn-Util → ...`. Behavior identical: every lib remains reachable as `Cairn.<Name>` via the namespace metatable. The internal sentinel metatable key was renamed from `__cairn_umbrella_installed` to `__cairn_namespace_installed`; Core.lua checks AND sets both so any older Cairn-Util (MINOR <= 14) loaded alongside still skip-installs correctly in either direction. - **`Cairn-Util.ObjectPool` owns the reset path; doesn't trust Blizzard's `disallowResetIfNew`.** Consumer convention is `creationFn = function() return obj end` and `resetFn = function(obj) end` — Cairn-style, no leak of the underlying Blizzard pool. The shim wraps `creationFn` only (so Blizzard's `(pool)` arg doesn't reach the consumer) and does NOT register `resetFn` with `CreateObjectPool` at all. Instead the Cairn lib holds the reset callback itself and fires it from its own `:Release`. Rationale: an earlier version of this fix set `disallowResetIfNew = true` on the Blizzard pool, which was supposed to suppress reset-on-create; the in-game smoke surfaced that the flag has no effect on current retail (reset still fired on Acquire-of-new, so `Fresh object is dirty` FAILed and `Reset count after one release` was off-by-one). Owning the reset path also gives deterministic ordering (reset fires before return-to-inactive, every time) and applies to `:ReleaseAll` too (which now snapshots active and calls `:Release` per object instead of going through Blizzard's reset-less `:ReleaseAll`). If you need the Cairn pool inside `creationFn`, capture it via upvalue from outside the `:New` call. Cairn-Util MINOR bumps 28 → 30 (two related changes in one batch).

### Fixed

- **`Cairn-Util.Hash.Combine` was order-dependent and didn't dedup.** The N-arg fold called `bit.bxor(result, select(i, ...))`, and bare `select(i, ...)` in a function-call position expands to ALL remaining args (`bit.bxor(result, args[i], args[i+1], ..., args[n])`). The suffix got folded into `result` on every iteration, so `Combine(100, 200)` and `Combine(200, 100)` returned 100 vs 200 (order-dependent), `Combine(42, 42)` returned 42 not 0 (dup-NOT-removing), and `Combine(1, 2, 3)` returned 2 not 0 (three-arg drift). Fix: force single-value capture via `local arg = select(i, ...)` before the xor. WHY-comment added on the line. Caught by the new CairnDemo smoke runner; previously masked because Forge_Console (which would have run the paste-in smoke) was broken against Cairn v14. Cairn-Util MINOR bumps 27 → 28.

- **`Cairn-Util.Table.MergeDefaults` clobbered user scalars when defaults supplied a table at the same key.** Previous logic: `if type(v) == "table" then if type(target[k]) ~= "table" then target[k] = {} end; recurse`. That branch fired even when `target[k]` was a non-nil scalar (e.g., user has `count = 5` and defaults add `count = { wrap = "table" }`), replacing the user's 5 with `{}` then merging defaults in -- the opposite of the documented intent ("surprise data loss if a default schema gets revised mid-flight"). Fix: three-way branch -- if `target[k] == nil`, create `{}` and recurse; if `target[k]` is already a table, recurse; otherwise preserve user value. Same MINOR bump (27 → 28).

- **`Cairn-Flow.FSM` sub-namespace shipped.** Third of four. Hierarchical state machine (HSM, not flat FSM) — composite states declare an `initial` child and their own `states` subtree; the machine is always "in" one leaf and that leaf plus every ancestor are simultaneously active. Public API: `Cairn.Flow.FSM:New({ initial = "X", states = {...} })`, `:Send(event, ...)`, `:GetState()` returns leaf name, `:GetPath()` returns root-to-leaf array, `:IsIn(name)` true for leaf or any ancestor, `:OnTransition(fn)` returns unsub closure. **Event dispatch** bubbles leaf-to-root: each state's `on[event]` is checked in turn; first match wins. Entry shapes are `EVENT = "Target"` (unconditional) or `EVENT = { target = "Target", guard = fn }` (conditional). **Decision B locked**: a guard returning false declines the transition and bubbling continues to the parent — a parent's `on[event]` can still match. **LCA-based transitions** (Harel local semantics): from old leaf walk up to but NOT including the LCA firing `onExit` at each; from one-below-LCA walk down to target firing `onEnter` at each; if target is composite, follow the `initial` chain into the deepest leaf firing `onEnter` at every level. LCA itself is neither exited nor entered (it stays active). **Open question (b) resolved**: `onExit` / `onEnter` receive `(event, ...)` where event is the event name (nil at the :New-time initial entry); `guard` receives `(...)` only (no event — guards are pure context predicates); `OnTransition(fn)` observers receive `(oldLeaf, newLeaf, event, ...)`. **Validation walks the full state tree at :New time** — initial references exist, all transition targets resolve to known states, no duplicate state names, composite states have both `initial` and `states`, `on` entries are well-shaped. **Error isolation**: every callback (onEnter, onExit, guard, observer) wrapped via `Cairn.Util.Pcall.Call`. A throwing guard is treated as a decline (transition declined, bubbling continues); a throwing onExit/onEnter does NOT abort the transition — the machine still ends up in the target state. A synthetic root wraps the consumer's top-level states internally so LCA always has a guaranteed common ancestor (stripped from `GetPath` output). ~365 LOC (vs the ~350 OBJECTIVES estimate). Reference: Miro Samek HSM design + David Harel statecharts. - **`Cairn-Flow.Decision` sub-namespace shipped.** Second of four. Table-declarative decision tree with two node shapes: **binary** (`{ test = fn, if_true = branch, if_false = branch }`) and **multi-way** (`{ value = fn, cases = { KEY = branch, ... }, default = branch }`). Branches resolve as: `nil` / `boolean` / `string` / `number` returned as-is; `function` called and its return value is the leaf (NOT recursed into — if you want recursion, embed a table directly); `table` recursed as a nested node. Passthrough args from `:Evaluate` reach every test / value / function-leaf in the tree — lets a single tree be reused across consumers without closure-baked state (poor man's blackboard). **Validation walks the full tree at `:New` time** so malformed structures surface up front rather than on first `:Evaluate` of a particular branch. **Error isolation** mirrors Sequencer: test/value/leaf fns wrapped via `Cairn.Util.Pcall.Call`; a throwing test takes `if_false`, a throwing value falls to `default`, a throwing leaf yields `nil`, all errors routed through `geterrorhandler`. ~120 LOC (vs the ~80 LOC OBJECTIVES estimate — the extra is the at-construction tree walker, error-isolation glue, and inline docs). Lives in `CairnUtil/Cairn-Util-String.lua`. Smoke test at `.dev/tests/cairn_util_string_smoke.lua` covers TitleCase across apostrophes / hyphens / spaces / digits / single-letters / idempotence and NormalizeWhitespace across CR/LF/tab stripping, edge trimming, internal collapsing, empty / whitespace-only / already-normal inputs (19 PASS/FAIL checks).

- **`Cairn-Addon` library.** Addon lifecycle wrapper providing `OnInit` / `OnLogin` / `OnDisable` handlers. The defining feature is **retro-fire**: if a consumer assigns `OnInit` or `OnLogin` after the matching event has already passed, the handler fires immediately. Fixes the main lifecycle pain point in the v1 library, where load-on-demand addons that registered late had their `OnLogin` silently never fire. LibStub MAJOR: `Cairn-Addon` (no version suffix). Smoke test at `.dev/tests/cairn_addon_smoke.lua`.
- **`Cairn-DB` library.** SavedVariables wrapper with default merging and named profiles. Surface: `New(name, defaults)`, `Get(name)`, `.instances`, plus instance fields `.global` / `.profile` and methods `:SetProfile(name)` / `:GetProfile()`. Trimmed compared to v1: no `realm` / `char` / `faction` sub-tables, no profile management methods beyond Set/Get — added only when a real consumer needs them. Unknown keys in `defaults` raise an error (catches typos loud). Defaults merge is non-destructive and runs on profile creation, so adding a new default key in a future release will fill into any newly-created profile. Smoke test at `.dev/tests/cairn_db_smoke.lua`.
- **`Cairn-Events` library.** Event routing through a single internal frame — handles BOTH WoW events (auto-routed via the frame) and internal addon-to-addon events (dispatched explicitly via `:Fire`). Surface: `Subscribe(event, handler, owner)` → subscription, `Unsubscribe(sub)`, `UnsubscribeOwner(owner)`, `Fire(event, ...)`, `.handlers`. Handler signature is `handler(...)` — no event-name prefix, no self. RegisterEvent failures are silent: an unknown event name is silently treated as internal (use `:Fire` to dispatch). Snapshot-during-dispatch lets handlers safely subscribe/unsubscribe mid-fire without iterator confusion. Handler errors are pcall-isolated; one bad handler doesn't break dispatch for the rest. No retro-fire — Cairn-Addon owns the lifecycle-event pattern; Cairn-Events is for ongoing routing. Use namespaced names like `"MyAddon:Ready"` for internal events to avoid clashing with WoW event names. Smoke test at `.dev/tests/cairn_events_smoke.lua`.

Renders every schema type via Cairn-Gui-2.0 widgets, including the storage-only types (text/color/keybind) that the Blizzard panel can't display. Falls back to a nil-with-warning if Cairn-SettingsPanel-2.0 isn't loaded.
- **`Cairn-Settings` library (flagship).** Declarative settings schema bridged to Blizzard's native Settings panel (the Dragonflight-era Settings API). Backed by a Cairn-DB instance the consumer provides — all values live in `db.profile`. Schema types: `header`, `toggle`, `range`, `dropdown` (rendered in the Blizzard panel); plus `text`, `color`, `keybind` (storage-only — schema validates, defaults seed, Get/Set/OnChange work, but the consumer renders their own UI for these because Midnight's Settings API broke the panel-button paths and we chose not to chase). Per-entry `onChange` callback PLUS instance-level `:OnChange(key, fn)` subscribers fire on Set, with an unsub closure returned for cleanup. Color defaults accept named (`{r=,g=,b=[,a=]}`) or positional (`{r,g,b[,a]}`) shapes; positional is normalized to named at validate time. Stub mode when the Blizzard Settings global is unavailable (older Classic flavors): Get/Set/OnChange still work; :Open prints a warning. Surface: `CS:New(name, db, schema) -> instance`, `instance:Get/Set/OnChange/Open/GetCategory/GetCategoryID`. Idempotent `LibStub("Cairn-Settings")(name, db, schema)` shorthand via `__call`. Weak-keyed `.instances` table for Forge_Registry introspection. v1 deliberately drops the v0.2 anchor type (needed Cairn-EditMode which isn't in the inventory) and `:OpenStandalone` (depended on Cairn-SettingsPanel-2.0 which lives in the original Cairn's GUI family). Both reintroduce-able when consumers materialize. Smoke test at `.dev/tests/cairn_settings_smoke.lua` covers schema validation (good shapes accepted, 8 bad shapes errored), default seeding, color positional normalization, Get/Set round-trip, onChange + subscribers + unsub, storage-only types, and Blizzard category registration.
- **`Cairn-Media-Icons` (sub-file of Cairn-Media).** Icon glyph registry living alongside Cairn-Media. Adds `:GetIconCodepoint(name)`, `:GetIconGlyph(name)` (returns UTF-8 string for direct SetText), `:HasIcon(name)`, `:ListIcons()`, `:IterIcons()`, and `:RegisterIcon(name, codepoint)` for consumer extension. Ships ~45 curated Material Symbols codepoints (navigation, actions, status, toggles, containers, WoW-flavored). Pair an icon glyph with one of the Material Symbols PUBLIC fonts (`MaterialOutlined` / `MaterialRounded` / `MaterialSharp`) registered in Cairn-Media's main file. Following the multi-file pattern established by Cairn-Util, Icons is a separate file (`Cairn-Media-Icons.lua`) that attaches to the Cairn-Media table at load time.
- **`Cairn-Media` library.** Two-mode asset registry for fonts, statusbars, borders, backgrounds, and sounds. Every entry is either PRIVATE (visible only to Cairn consumers via `:GetFont` etc.) or PUBLIC (also registered with `LibSharedMedia-3.0` so cross-addon dropdowns in WeakAuras / ElvUI / Details! / etc. pick it up). Lookup is uniform across both buckets — public wins on collision. Public registrations carry the `Cairn ` prefix in LSM so attribution is clear. Surface: `:GetFont` / `:GetStatusbar` / `:GetBorder` / `:GetBackground` / `:GetSound` for path lookup; `:ListFonts` etc. for arrays of names with optional `"public"` / `"private"` filter; `:ListPublicFonts` / `:ListPrivateFonts` sugar per type; `:Iter(type, visibility)` for sorted iteration; `:Has(type, name)` and `:IsPublic(type, name)` for inspection. WoW's built-in fonts, statusbars, borders, backgrounds, and sounds ship as PRIVATE entries under friendly names (`Default`, `Numeric`, `Heading`, `Combat`, `Plain`, `Solid`, `Tooltip`, etc.) — no binary files included; the lib just references Blizzard's canonical paths. PUBLIC bucket starts empty; consumers drop free-license assets under `CairnMedia/Assets/Public/<type>/` and add a `registerPublic` line. Vendors `LibSharedMedia-3.0` as a soft dep (`LibStub("LibSharedMedia-3.0", true)`) under `Libs/`. Smoke test at `.dev/tests/cairn_media_smoke.lua`.
- **`Cairn-Util` library.** A collection of small utilities organized into named sub-namespaces, each living in its own file. The main `Cairn-Util.lua` is a thin namespace skeleton; category files attach themselves to the lib's table at load time. Consumer view stays simple: `LibStub("Cairn-Util").Hash.MD5(...)`. Adding a category later is "drop a new `Cairn-Util-<Category>.lua` file and TOC line" with no main-lib surgery. v1 ships only the `Hash` sub-namespace; Math / String / etc. land here when consumers materialize. **Hash:** `Cairn-Util.Hash.MD5(input)` → 32-char lowercase hex digest, `Hash.MD5Raw(input)` → 16 raw bytes. Underlying implementation is the vendored `AF_MD5.lua` (kikito md5.lua + enderneko's WoW compatibility tweaks, MIT licensed — original copyright notice preserved in the vendored file). MD5 is intended for fingerprinting / identity / content-keyed caches — NOT cryptographic security; the docstring calls this out explicitly. Smoke test verifies four RFC 1321 test vectors plus internal hex↔raw consistency. Test file at `.dev/tests/cairn_util_smoke.lua`.
- **`Cairn-Callback` library.** CallbackHandler-1.0 compatibility shim. Registered under both `Cairn-Callback` and `CallbackHandler-1.0` LibStub MAJORs; the upstream-MAJOR registration is at MINOR=1 so any other CallbackHandler-1.0 implementation in the user's environment (ElvUI's bundled copy, AceComm's, etc.) wins the slot — we provide a working fallback for when nothing else is loaded. Public API mirrors upstream exactly: `:New(target, regName, unregName, unregAllName, OnUsed, OnUnused)` installs `RegisterCallback` / `UnregisterCallback` / `UnregisterAllCallbacks` on `target`, returns a registry with `:Fire(eventName, ...)`. Dispatch routes per upstream convention: function-form `fn(eventName, ...)`, method-string form `consumer:Method(eventName, ...)`, with optional arg prepended before eventName in both forms. Snapshot-during-Fire so handlers can safely unregister themselves or peers mid-dispatch. Handler errors are pcall-isolated and routed to `geterrorhandler`. `.instances` table exposes all registered callback targets for Forge_Registry introspection. Smoke test at `.dev/tests/cairn_callback_smoke.lua` (covers function/method/with-arg dispatch, OnUsed/OnUnused, self-unregister during fire, error isolation, and a round-trip through the `CallbackHandler-1.0` shim).
- **`Cairn-Timer` library.** Timing primitives with ownership tracking. `:After(delay, fn, opts)` for one-shot, `:Every(delay, fn, opts)` for repeating (opts.count for "fire N times then auto-cancel" — no consumer-side counter bookkeeping), `:Debounce(key, delay, fn, opts)` to collapse rapid calls keyed by string (only the final call after `delay` of quiet fires the handler), and `:Stopwatch()` for elapsed-time measurement with named laps. Handle methods: `:Cancel`, `:IsCancelled`. Stopwatch methods: `:Read`, `:Lap`, `:Laps`, `:Stop`, `:Reset`, `:IsStopped`. Batch cancel via `:CancelOwner(owner)`. Callback errors are pcall-isolated and routed to `geterrorhandler`; a throwing tick of an `Every` does NOT stop subsequent ticks. Smoke test at `.dev/tests/cairn_timer_smoke.lua` (split sync + async; async uses Forge_Console build 13+ deferred-print streaming).
- **`Cairn-Hooks` library.** Pre/Post/Wrap hook helpers for instrumenting addon functions. API: `:Pre(target, method, fn, owner)`, `:Post(target, method, fn, owner)`, `:Wrap(target, method, fn, owner)` all return a hook handle; `:Unhook(handle)`, `:UnhookOwner(owner)` for cleanup; `._registry` flat array for introspection. Pre runs before the original; Post runs after with original args (return values pass through unchanged, including multi-returns and embedded nils); Wrap replaces the original and receives it as the first arg so the consumer decides whether/how to invoke it. Multiple hooks compose — last-installed wraps outermost. Pre/Post errors are pcall-isolated and routed to `geterrorhandler`; Wrap errors propagate (Wrap is the "I take responsibility" mode). Last unhook restores the original function as-is (no leftover wrapper). Note: wrapping a Blizzard secure function will taint it; use Post (or `hooksecurefunc` directly) for secure post-hooks. Smoke test at `.dev/tests/cairn_hooks_smoke.lua`.
- **`Cairn-Log` library.** Shared categorized ring-buffer log. One source-level logger per addon via `LibStub("Cairn-Log"):New("MyAddon")`, optional category sub-loggers via `:Category("net")`. Logging methods are `:Debug`, `:Info`, `:Warn`, `:Error`, and `:Log(level, ...)` for custom levels. Args use `string.format`; passing a single arg skips formatting so callers don't have to escape `%`. The entire process shares one ring buffer (default capacity 1000, configurable via `:SetCapacity`); Forge_Logs reads from it via `:GetEntries(filter)` with support for source / category / level / since / limit filters. `:SetChatEchoLevel(level)` opts into chat echoing for entries at or above the threshold (default silent). Entry shape: `{ timestamp, source, category, level, message }`. Persistence is out of scope — buffer is in-memory and dies on /reload; layer SavedVariables on top via Cairn-DB if needed. Smoke test at `.dev/tests/cairn_log_smoke.lua`.
- **`Cairn-Locale` library.** i18n table with locale fallback. One instance per addon: `LibStub("Cairn-Locale"):New("MyAddon")`. Surface: `:Set(locale, strings)` (merges), `:Get(key)`, plus `L[key]` metatable-based lookup as a shorthand. ALL registered locales stay in memory simultaneously — language switching at runtime is a single lookup change, no `/reload` required. Lookup fallback chain: current locale → enUS → the key itself (never returns nil, so consumers don't have to guard for missing translations). Lib-level `SetOverride(locale)` / `GetLocale()` for dev-tool locale switching (e.g. Forge `/forge locale deDE`); when the effective locale changes, fires `Cairn-Locale:Changed` via Cairn-Events with `(newLocale, oldLocale)` so UIs can subscribe and refresh. Cairn-Events is a soft dep — event is skipped silently if it isn't loaded. Methods take priority over locale keys named the same — `L.Set` always resolves to the method; if you genuinely have a string keyed `"Set"`, use `L:Get("Set")`. Recommended file structure for shareable translation work: one file per language (`Locales/enUS.lua`, `Locales/deDE.lua`, ...), each calling `:New(name)` (idempotent) + `:Set(locale, strings)`. Order in the TOC doesn't matter. Smoke test at `.dev/tests/cairn_locale_smoke.lua`.
- **`Cairn-Slash` library.** Slash command registry with **nested** subcommand routing and recursive auto-help. Surface: `Register(name, slash, opts)` → root node, `Get(name)`, `.registry`. Every node (root and every sub) is the same shape and supports the same methods: `:Sub(name, handlerOrDescription, description)`, `:Default(handler)`, `:GetSubs()`. `:Sub` is overloaded by type of the second arg — function = leaf (handler is set), string = group description (no handler), nil = bare group. `:Sub` returns the created sub so you can chain or attach further nested subs to it. `:Default(handler)` sets the calling node's handler at any depth — on root it's "what runs when nothing matches /slash"; on a sub it's "what runs at /slash sub when no further nested match". Dispatch walks the tree as far as msg's leading tokens match, then calls the deepest matched node's handler with the unmatched remainder. When a node has no handler, the auto-help fallback **recursively walks every descendant and prints full slash paths** so the user sees the complete tree, not just the immediate level. Handler errors are pcall-isolated. Smoke test at `.dev/tests/cairn_slash_smoke.lua`.
- **`Cairn-Util.Pcall` sub-namespace.** `Pcall.Call(context, fn, ...)` is the new shared pcall wrapper. Routes failures through `geterrorhandler` with the format `"<context> threw: <err>"`. Returns `(ok, err_or_returns)` so dispatch loops can branch on success when needed.
- **`Cairn-Util.Table` sub-namespace.** `Table.Snapshot(arr)` produces a shallow array copy for safe iteration when the source may mutate mid-loop (handler self-unsubscribe, timer scheduling a new timer in the same bucket, etc.). `Table.MergeDefaults(target, defaults)` recurses for tables and preserves existing leaves (user data wins; scalar wins over a defaults-side table to avoid surprise data loss).

Lives at `SubAddons/CairnDemo/`. `/cairndemo run` executes a 16-item PASS/FAIL smoke covering Addon / DB / Events / Slash / Locale / Log / Hooks / Timer / Callback / Util / Media / Settings + namespace resolution. ### Changed

- **`Cairn-Gui-2.0` consumes the v1.0 lib surface (not the old v1 API).** Two API drifts in the verbatim port were fixed at the consumer side rather than shimmed in the libs: (1) `Theme-Default-2.0` now calls `Cairn-Media:GetFont(name)` instead of the old `GetFontPath(name)`; (2) `Core/Events.lua` no longer depends on Cairn-Callback — its per-widget pub/sub is an inline ~25-line registry. Cairn-Callback stays a pure CallbackHandler-1.0 upstream-compat shim with no extra surface, and Cairn-Media keeps its current Get* method names.
- **`_G.Cairn` namespace restored.** Cairn-Util installs a metatable on `_G.Cairn` whose `__index` resolves `Cairn.<Name>` to `LibStub("Cairn-<Name>-1.0")` lazily. Consumers can write `Cairn.Addon:New("MyAddon")` instead of `LibStub("Cairn-Addon-1.0"):New("MyAddon")`. Existing v1 consumers that used the `_G.Cairn` table will see their `Cairn.X` lookups continue to resolve. Idempotent across reloads, chains any prior `__index`, and rawset entries (e.g. `Cairn.LogWindow` installed by `Cairn-LogWindow-2.0.lua`) take precedence. Gui libs are NOT in the namespace (they're `-2.0` MAJORs); use `LibStub("Cairn-Gui-2.0")` directly for Gui.
- **LibStub MAJORs now carry a `-1.0` suffix; all MINORs reset to 14.** Every rewritten lib's LibStub MAJOR gains the conventional version suffix (`Cairn-Addon` → `Cairn-Addon-1.0`, `Cairn-DB` → `Cairn-DB-1.0`, etc. across all 12: Addon / DB / Events / Slash / Locale / Log / Hooks / Timer / Callback / Util / Media / Settings). Every lib's `LIB_MINOR` was set to **14** so this build numerically supersedes anything previously published under the bare-MAJOR names. Consumers must update their `LibStub("Cairn-Addon")` calls to `LibStub("Cairn-Addon-1.0")` and so on. The Cairn-Gui family (`Cairn-Gui-2.0` and its sub-MAJORs) is unaffected — it keeps its `-2.0` suffix. The `CallbackHandler-1.0` upstream-compat shim slot is also unchanged. Smoke tests, README, and lib-internal doc snippets were swept to match.
- **Load order: `Cairn-Util` now loads first.** TOCs moved the `CairnUtil/` block immediately after vendored `LibStub`, so every other Cairn lib can pull shared helpers via `LibStub("Cairn-Util-1.0")`. Lets us delete duplicated `safeCall` / `mergeDefaults` / snapshot-loop copies from individual libs without changing their public behavior.
- **Folder renamed: `Cairn-Gui-2.0/` → `Cairn-Gui/`.** The Cairn-Gui family's top-level folder is now just `Cairn-Gui/`. Its sub-folders (`Cairn-Gui-Widgets-Standard-2.0/`, `Cairn-Gui-Widgets-Secure-2.0/`, `Cairn-Gui-Layouts-Extra-2.0/`, `Cairn-Gui-Theme-Default-2.0/`) keep their `-2.0` names because they match the LibStub MAJORs they register. LibStub MAJORs themselves are unchanged (`Cairn-Gui-2.0`, `Cairn-Gui-Widgets-Standard-2.0`, …); this is a filesystem rename only. TOC paths updated accordingly.
- **Pcall, Snapshot, MergeDefaults extracted into Cairn-Util.** Seven libs (`Cairn-Addon`, `Cairn-DB`, `Cairn-Events`, `Cairn-Slash`, `Cairn-Hooks`, `Cairn-Timer`, `Cairn-Callback`) previously carried their own near-identical `safeCall` / snapshot-during-dispatch / `mergeDefaults` copies. They now consume the shared `Cairn-Util.Pcall.Call` / `Cairn-Util.Table.Snapshot` / `Cairn-Util.Table.MergeDefaults`. Lib-level error message format is unchanged byte-for-byte (still `"Cairn-Foo: <label> threw: <err>"`); each lib keeps a thin local wrapper that builds the lib-specific context string and delegates to `Pcall.Call`. `Cairn-Locale` and `Cairn-Log` were audited and skipped — Locale uses no pcall, and Log's only pcall has a different contract (return a fallback string on format error, never route to errorhandler). Per-lib MINORs bumped: Addon 1→2, DB 1→2, Events 1→2, Slash 2→3, Hooks 1→2, Timer 1→2, Callback 1→2.
